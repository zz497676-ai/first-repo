# 王者荣耀选人辅助（第一期）

选人（BP）阶段的实时辅助工具：手动把双方选的英雄点进选人板，页面会实时计算克制关系、英雄强度、阵容合理度，并给出一个粗略的胜率预测。

## 这一期做了什么

- **手动选人 UI**：像游戏选人界面一样，点击空位从英雄列表里选人 / ban 人。截屏/录屏自动识别选人界面是下一期功能，这一期没有做，但选人这一步被抽成了单一的 store action（`assignHero`），以后接自动识别时不需要改这层。
- **实时推荐**：根据当前的 ban/pick 状态，给己方下一手推荐候选英雄，附强度/克制/阵容三部分打分依据。
- **阵容合理度**：规则式打分（定位覆盖、伤害类型混合、开团/控制/保护），返回分数和具体问题提示。
- **胜率预测**：把强度差、净克制优势、阵容分差加权算成一个百分比，仅供参考。

纯前端静态页面，没有后端、没有网络请求、没有账号系统——所有计算都是对本地 JSON 数据做的即时运算。

## 关于数据

`src/data/heroes.json` 和 `src/data/counters.json` 是**示例/近似数据**，不是官方数据：

- 英雄强度评分、克制关系都是根据英雄技能机制估计的示例值，标注了 `disclaimer` 字段。
- 目前只精选了 24-28 个常见英雄（覆盖六个定位），克制关系也只列了几十条典型的，没有覆盖全部英雄和全部两两组合——没写的英雄对默认视为中立。
- 想要更新/扩充数据，直接编辑这两个 JSON 文件即可（`Hero` / `CounterEntry` 的字段说明见同目录下的 `.types.ts` 文件），改完 `strengthScore` 记得让 `tier` 保持一致（`heroes.types.ts` 里的 `tierFromScore` 有分档规则，`tests/data/dataIntegrity.test.ts` 会校验这个一致性）。

## 开发

```bash
npm install
npm run dev       # 本地开发，默认 http://localhost:5173
npm run test      # 跑一遍 Vitest 单测
npm run build     # 类型检查 + 生产构建
```

## 目录结构

- `src/data/` — 英雄和克制关系的静态 JSON 数据 + 类型定义
- `src/engine/` — 纯函数打分引擎（克制分数、阵容评分、胜率预测、推荐排序），可单测
- `src/store/` — Zustand 状态管理，选人板的唯一数据源
- `src/components/` — 选人板、英雄选择器、推荐面板、阵容仪表、胜率条
- `tests/` — 引擎单测 + 数据完整性校验
